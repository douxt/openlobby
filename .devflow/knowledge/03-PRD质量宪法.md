# PRD 质量宪法（11 项）

> 每次生成、修改 PRD 时，agent 必须逐条对照以下 11 项。缺任一项的 PRD 不得进入 Gate 3。
>
> 基于 7 个 PRD 阶段踩坑 + SafeBreach / SDD / EARS / 100 分评分卡等外部实践提炼。

**使用方式**：调用 `/to-prd` 时附带本文件作为上下文，agent 逐条自检后在 PRD 尾部输出合规表。

## 完整性与结构

| # | 规则 | 核心要求 | 违规后果 |
|---|------|---------|----------|
| 1 | **章节完整** | Problem / Solution / User Stories / Implementation / Testing / Out of Scope 六段必含 | AFK agent 缺少实现指引 |
| 2 | **Risks & Mitigations** | ≥5 项风险，各有缓解措施 | 后期返工补风险分析 |
| 3 | **定量成功标准** | AC 可测量，禁止模糊词；推荐 EARS 格式 | 验收标准不可验证 |
| 4 | **异常路径覆盖** | 空状态 / 错误状态 / 边界条件 / 权限拒绝 4 类 | Agent 自行脑补边界行为 |

## 外部依赖与接口

| # | 规则 | 核心要求 | 违规后果 |
|---|------|---------|----------|
| 5 | **接口签名明确** | 外部依赖写到函数签名/参数/返回值级 | Agent curl 探索 API |

## 内部一致性

| # | 规则 | 核心要求 | 违规后果 |
|---|------|---------|----------|
| 6 | **内外一致性** | 各章节无矛盾；尾部列关联文档同步清单 | 实现时发现矛盾回退 |

## 可执行性

| # | 规则 | 核心要求 | 违规后果 |
|---|------|---------|----------|
| 7 | **工作量估算 ≤5d** | 单 PRD ≤5d，超出则拆分 | AFK 上下文超限 |
| 8 | **User Story 可验证** | "As a...I want...so that" 格式，有独立 AC | US 不可测试 |
| 9 | **Out of Scope 不含未来期** | 不含 Phase N+1 内容 | 范围蔓延 |

## 技术决策

| # | 规则 | 核心要求 | 违规后果 |
|---|------|---------|----------|
| 10 | **技术选型有理由** | 每个技术决策有 justification | 随意选型后期推翻 |
| 11 | **架构约束已引用** | 显式列出不可变架构规则 | 实现违反架构设计 |

## N/A 豁免

- **#8**：纯技术 PRD 不含 User Stories 时可标 N/A
- **#5**：无外部依赖的 PRD 可标 N/A

## 合规表模板

agent 在 PRD 尾部输出：

```markdown
## PRD 质量宪法合规表

| # | 规则 | 状态 | 证据/位置 |
|---|------|:----:|----------|
| 1 | 章节完整 | ✅ | §Problem / §Solution / §US / §Impl / §Test / §OOS |
| 2 | Risks ≥5 + 缓解 | ✅ | §Risks 共 7 项 |
| 3 | 定量 AC | ✅ | §AC 全部量化 |
| 4 | 异常路径 4 类 | ✅ | §Edge Cases |
| 5 | 接口签名 | ✅ | §External APIs |
| 6 | 内外一致 | ✅ | 各章节数据对齐 |
| 7 | ≤5d | ✅ | §Estimate: 3d |
| 8 | US 可验证 | ✅ | US-001~US-005 各有 AC |
| 9 | OOS 无 N+1 | ✅ | §Out of Scope 仅含 Phase 1 |
| 10 | 技术选型有理由 | ✅ | §Decisions 每条有 justification |
| 11 | 架构约束已引用 | ✅ | §Constraints 引用 R1-R5 |
```
