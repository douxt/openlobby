# 脚本质量宪法（12 项）

> 生成/修改调度器、prep 脚本、CI 脚本时，逐条对照以下 12 项。缺任一项的脚本不得上线运行。

## 运行时安全

| # | 检查项 | 规则 | 违规后果 |
|---|--------|------|----------|
| 1 | **shell 选项** | 入口设置 `set -euo pipefail`，或显式关闭（`set +e` 特定段落） | grep 无结果静默崩溃 |
| 2 | **grep/awk 空结果防护** | 所有 `$(grep ... | awk ...)` 赋值末尾加 `|| true` 或 `|| echo ""`；空敏感判断用 `grep -q` 单独处理 | 脚本静默退出 |
| 3 | **ZSH_VERSION 兼容** | 入口加 `: "${ZSH_VERSION:=}"` 防 Claude Code hook wrapper 崩溃 | set -u 下 unbound variable |
| 4 | **变量作用域** | 所有被 guard 后的分支引用的变量，定义在 guard 之前；`set -u` 下不允许"一个分支定义另一个分支引用" | worker 模式报 unbound variable |

## 健壮性

| # | 检查项 | 规则 | 违规后果 |
|---|--------|------|----------|
| 5 | **trap 清理** | `trap cleanup EXIT INT TERM`，cleanup 函数 kill 后台进程 + 回退残留 issue 状态 + prune worktree | 中断后 issue 卡在 in_progress |
| 6 | **日志安全** | 不直接 `exec > >(tee ...)` 做全量日志；用独立 `log()` 函数写入，或 trap 中显式 `wait` 保 flush | Ctrl+C 丢日志 |
| 7 | **worktree 隔离** | 每个 worker 使用独立 git worktree；同一时间不共享 worktree；结束后 `git worktree prune` | 文件冲突 |
| 8 | **幂等性** | 脚本可中断重跑，不会因残留状态（in_progress/ worktree）报错 | 重跑需要手动清理 |

## 权限与隔离

| # | 检查项 | 规则 | 违规后果 |
|---|--------|------|----------|
| 9 | **per-worker 权限** | 多 worker 并行时，每个 worker 的 settings deny 除自身 issue 外所有其他 issue 文件读权限 | worker 读错 issue 实现错模块 |
| 10 | **非 root** | 禁止 `sudo`、禁止全局 `chmod`、禁止修改 `/etc/` | 安全违规 |

## 可控性

| # | 检查项 | 规则 | 违规后果 |
|---|--------|------|----------|
| 11 | **显式 error handling** | 不依赖 `set -e` 兜底；关键步骤（git push、worker 执行）有 `if ! cmd; then` 独立错误处理 | 半失败状态无法定位 |
| 12 | **重试上限** | 同一操作最多 3 次重试，超出输出 `[UNRESOLVED]` | 死循环消耗 token |
