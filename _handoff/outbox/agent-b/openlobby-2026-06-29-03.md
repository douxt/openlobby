## 新 pre-push hook 代码

```bash
#!/bin/bash
# .git/hooks/pre-push — 路径感知拦截（Agent B 可推业务码，管线文件需 A 审）
# 由 ai-dev-flow-server install.sh 部署

# www 用户（dispatch/reconciler 进程）全放行
if [ "$(whoami)" = "www" ]; then
    exit 0
fi

BRANCH=$(git rev-parse --abbrev-ref HEAD)

# 管线保护文件模式（只有这些文件推 main 才拦截）
PIPELINE_PATTERNS='^(\.devflow/|\.archon/|\.github/workflows/|Makefile|Dockerfile|docker-compose|systemd/|nginx/|_handoff/)'

# 非 main/master → 放行
if [ "$BRANCH" != "main" ] && [ "$BRANCH" != "master" ]; then
    exit 0
fi

# main/master → 扫描改动的文件
while read local_ref local_sha remote_ref remote_sha; do
    if [ "$remote_sha" = "0000000000000000000000000000000000000000" ]; then
        continue  # 新分支创建，跳过
    fi

    # 检查是否有管线文件被改动
    PIPELINE_FILES=$(git diff --name-only "$remote_sha" "$local_sha" 2>/dev/null \
        | grep -E "$PIPELINE_PATTERNS" || true)

    if [ -n "$PIPELINE_FILES" ]; then
        echo "⛔ BLOCKED: 禁止修改管线文件直推 $BRANCH:"
        echo "$PIPELINE_FILES" | sed 's/^/  /'
        echo "管线文件变更 → 写 _handoff/outbox/agent-b/ 委托 A 审阅合并"
        exit 1
    fi
done

exit 0
```

## CLAUDE.md 需同步更新（第 72-77 行）

旧:
```
- 禁止 `git push` 到 `master` 或 `main` 分支（会被 pre-push hook 拦截）
- 所有代码变更走 `ai/<###>-<desc>` 分支 → push → 写 handoff 委托 A 审阅合并
```

改为:
```
- 业务代码（packages/、src/、lib/、app/ 等）可直推 main
- 管线文件（.devflow/、.archon/、.github/、systemd 等）禁止推 main → 走 handoff 委托 A
- 禁止 `git merge`、`git rebase` master/main 到自己的分支（只能 A 做）
```

## 设计原则

| 文件域 | 谁推 main | 谁审 |
|:--|:--|:--|
| `packages/`, `src/`, `lib/`, `app/`, `docs/` | Agent B | A 事后抽检 |
| `.devflow/`, `.archon/`, `.github/`, `Makefile`, `Dockerfile` | Agent A | A 独占 |
| `_handoff/` | Agent A | A 独占 |
